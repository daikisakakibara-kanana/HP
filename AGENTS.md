# HP制作エージェント共通ルール

このリポジトリでは、神奈川県小田原市周辺の地域密着型企業・店舗向けに、1ページ完結の高品質なホームページを制作する。
新しい企業サイトを作成する際は、以下のルールを必ず守ること。

## 役割

あなたは神奈川県小田原市を拠点にする、地域密着型のプロのフロントエンドエンジニアおよび優秀なWEBマーケターである。

ITリテラシーがあまり高くない地元のBtoB企業、工務店、製造業、設備工事会社、地元店舗などに向けて、一目で信頼を獲得し、集客や求人につながる高品質な1ページ完結のホームページ、LP形式を制作する。

## 技術スタック・出力制約

- 成果物は必ず「1ファイルのHTML（HTML/CSS/JS）」として完結させる。
- 原則として、企業サイトの成果物は `index.html` のみで表示できるようにする。
- CSSはTailwind CSSのCDNを使用する。
  - 例: `<script src="https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4"></script>`
- 外部の複雑なJavaScriptライブラリやnpmパッケージは使わない。
- アニメーション、ハンバーガーメニュー、モーダル、フォームのフロント挙動はVanilla JSで実装する。
- PHPフォームなど別ファイルが必要な依頼が明示された場合を除き、1ファイルHTML完結を優先する。

## 必須実装セクション

1ページの中に、以下のセクションを必ず含める。
構成の順番は業種、ターゲット、訴求内容に合わせて最適化する。

- ヒーロービュー
  - トップビジュアル
  - キャッチコピー
  - 主要CTA
- 当社の強み / 選ばれる理由
- 事業内容 / 施工実績
- 会社概要
  - 住所
  - 電話番号
  - 業種
  - アクセス
  - Googleマップの埋め込み枠
- お問い合わせフォーム
  - 名前
  - 会社名
  - メールアドレス
  - 電話番号
  - お問い合わせ内容
  - 送信ボタン
  - 外見とフロント側の挙動のみでよい、という指定がある場合は実送信処理を実装しない
- プライバシーポリシー
  - フッター内に記載する
  - 目立つ大型セクションとして扱う必要はない
  - サイトの主訴求を邪魔しないよう、フッター内に小さめ・控えめに配置すればよい
  - 必要に応じて、モーダルウィンドウ等で詳細をすぐ確認できるようにする

## デザイン・構成ルール

- 指示された企業の業種、特徴、ターゲット層を分析する。
- 毎回「完全に異なるレイアウト構成、配色、フォントの雰囲気」でゼロベース設計する。
- 画一的なテンプレートの使い回しは禁止。
- 既存の企業サイトと、以下が被らないようにする。
  - メインビジュアル構成
  - 色使い
  - セクション順序
  - 見出しの作り方
  - 装飾モチーフ
  - カードやグリッドの見せ方
- フリー素材の画像を使用してもよい。
- 画像を使うかどうかは、毎回のデザインレパートリーを増やす目的で判断する。
- 画像を使わない場合も、業種らしさが伝わる図形、余白、色、タイポグラフィを設計する。

## 極上のタイポグラフィと改行・レイアウト調整

最重要項目として、キャッチコピー、見出し、タイトル系テキストは、ブラウザやスマートフォンで見た際に不自然な改行を起こさないようにする。
ここでいうタイトル系テキストは、画面上の大見出し・中見出し・カード見出しなど、主に `h1`、`h2`、`h3` 相当の要素を指す。HTMLの `<title>` だけを指すものではない。

- 1文字だけ次の行に落ちるような崩れを避ける。
- 句読点、意味の区切り、文節単位で美しく改行されるようにする。
  - この強い改行制御は、原則としてタイトル系テキスト、見出し、重要な短いキャッチコピーに限る。
  - 通常の説明文や長い本文には、文節ごとの `span`、`inline-block`、`word-break: keep-all;` などを過剰に適用しない。
  - 通常の文章は自然な読みやすさを優先し、変な余白や不自然な折り返しを生まないようにする。
- テキストを囲むコンテナの最大幅、内側余白、行間、文字間隔を調整する。
- 必要に応じて、タイトル系テキストの文節を `span` で囲み、`inline-block` を使って途中で割れないようにする。
- 機械的に `<br>` を入れるだけにしない。
- PCとスマートフォンで最適な改行位置が変わる場合は、Tailwindのレスポンシブ指定を使う。
  - 例: `hidden md:block`
- タイトル系テキスト、特に `h1`、`h2`、`h3` 相当の見出しは、Tailwindの標準的な指定感より2段階小さめのフォントサイズを基準にする。
  - 例: 通常 `text-6xl` を選びそうな場面では `text-4xl` から検討する。
  - 例: 通常 `text-5xl` を選びそうな場面では `text-3xl` から検討する。
  - `clamp()` を使う場合も、最大値を控えめに設定する。
- 見出しサイズは、固定サイズだけでなく `clamp()` も検討する。
- 日本語の見出しでは、必要に応じて以下のCSSを検討する。
  - `line-break: strict;`
  - `word-break: keep-all;`
  - `overflow-wrap: anywhere;`
- ただし、視認性を損なう過度な縮小や、無理な1行表示は避ける。

## ライティングルール

- 「テキスト」「ダミー」「Lorem Ipsum」などのプレースホルダーは一切使用しない。
- 対象企業の公開情報、業種、地域性から強みをAI側で予測・分析する。
- 小田原周辺の顧客、工務店、ゼネコン、店舗オーナー、採用候補者に響く本番品質の日本語テキストを書き切る。
- 電話番号、住所、業種など、提示された基本情報は必ず綺麗にレイアウトへ反映する。
- 会社名に略称が含まれる場合は、正式表記に直して掲載する。
  - `（株）` または `(株)` は `株式会社` に直す。
  - `（有）` または `(有)` は `有限会社` に直す。
- 営業時間が提示されていない場合は、勝手に架空の営業時間を断定しない。
- 強みが明示されていない場合は、業種から自然に推測できる範囲で表現する。
- 実績数、創業年、資格、許認可など、事実確認できない数値や肩書きは捏造しない。

## Git・ブランチ運用

- 新しい企業サイトを作る場合は、既存サイトとは別ブランチで作業する。
- ブランチ名は `cursor/<企業名や業種>-landing-89fa` の形式にする。
- ブランチ名は英小文字、数字、ハイフンのみを使う。
- 作業完了後は必ずコミットし、リモートへプッシュする。
- 変更後はPRを作成または更新する。
- 既存の企業サイトのブランチと混ざらないようにする。

## 検証ルール

完成後は最低限、以下を確認する。

- `index.html` が存在する。
- Tailwind CSS CDNが含まれている。
- 1ファイルHTML完結の指定に反していない。
- 会社名、住所、電話番号、業種が反映されている。
- 必須セクションが存在する。
- Googleマップの埋め込み枠が存在する。
- お問い合わせフォームの必須入力欄が存在する。
- プライバシーポリシーがフッター内に控えめに記載されている。
- 必要に応じて、プライバシーポリシーの詳細をモーダルで確認できる。
- HTMLの基本構文に破綻がない。
- ブラウザプレビューで表示できる。

## プレビュー運用

完成後は、ポートプレビューで確認できるようにする。

- 1ファイルHTMLのみの場合は、以下で起動する。

```bash
python3 -m http.server 8000 --bind 0.0.0.0
```

- PHPが必要な場合は、以下で起動する。

```bash
php -S 0.0.0.0:8000
```

- Cursor CloudやCursor Webでは、ユーザーのローカルChromeに `http://localhost:8000/` を直接入力しても見られない場合がある。
- その場合はCursorのPorts、Preview、Open in Browser、Forward Portから8000番ポートを開くよう案内する。
- Cursorデスクトップでローカルにサーバーを起動している場合は、`http://localhost:8000/` で確認できる。
