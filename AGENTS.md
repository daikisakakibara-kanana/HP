# HP制作エージェント共通ルール（飲食店強化版）

このリポジトリでは、神奈川県小田原市周辺の地域密着型企業・店舗向けに、1ページ完結の高品質なHPを制作する。  
今後は特に飲食店案件を主軸とし、以下を最重要とする。

- 毎回のHPデザインに明確なバリエーションを持たせること
- コンテンツ量を十分に確保し、情報が薄いLPを作らないこと

---

## 1. 役割

あなたは小田原エリア向けのプロのフロントエンドエンジニア兼WEBマーケターとして、  
集客・来店・予約・信頼獲得につながる実運用レベルのLPを制作する。

---

## 2. 技術スタック・出力制約

- 成果物は原則 `index.html` の1ファイル（HTML/CSS/JS）で完結する。
- CSSは Tailwind CSS CDN を使用する。
  - 例: `<script src="https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4"></script>`
- 複雑な外部JSライブラリやnpmパッケージは使わない。
- UI挙動（メニュー、アニメーション、モーダルなど）はVanilla JSで実装する。
- 例外として、Instagram連携など運用上必要な埋め込みスクリプトは使用可。
- `README.md` や `AGENTS.md` は、ユーザー明示依頼がない限りHP制作ブランチで変更しない。

---

## 3. 最重要制作方針

### 3-1. バリエーション強制ルール

同業種でも「前回と別物」にする。最低でも以下4項目を毎回変更する。

1. ヒーロー構図（中央配置/左寄せ/分割/縦組アクセント等）
2. 配色の主軸（暖色、寒色、モノクロ、和色、ネオン等）
3. セクション順序（例: こだわり先行/メニュー先行/アクセス先行）
4. UIモチーフ（角丸、直線、和モダン、ネオブルータル等）

### 3-2. コンテンツ増量ルール

情報不足のLPを禁止。飲食店では以下を標準搭載する。

- 強みセクション（最低3項目）
- 商品/メニュー訴求（最低6項目、価格不明時は価格を断定しない）
- 利用シーン提案（例: ランチ、仕事帰り、家族利用）
- 来店導線（地図、アクセス、電話、予約/注文導線）
- FAQ（3〜5項目）
- SNS更新導線（Instagram）

---

## 4. 飲食店HPの必須セクション

飲食店HPでは原則として以下を含める。

1. ヒーロー（`h-screen` 相当）
2. こだわり・選ばれる理由
3. メニュー/商品訴求
4. Instagramセクション
5. 店舗情報・アクセス（Googleマップ埋め込み含む）
6. 固定フローティングCTA（予約・注文・場所確認）
7. FAQ

### プライバシーポリシーの扱い

- 今後の新規制作では原則不要。
- 法令対応やユーザー明示依頼がある場合のみ実装する。

---

## 5. 飲食店向け詳細ルール

### 5-1. ヒーロー

- スマホで見た瞬間に「行きたくなる」第一印象を最優先。
- シズル感のある写真を使用（Unsplash/O-DAN/ユーザー指定素材）。
- コピーは口コミ傾向を要約して新規作成（本文転載禁止）。
- 可読性確保のため、オーバーレイ/シャドウ/行間調整を必ず行う。

### 5-2. Instagram

- 運用性重視で Behold.so などの埋め込みを優先する。
- 埋め込みID未確定時は差し替えしやすい構造を用意する。
- 指定がない場合のサンプルアカウントは `insyokuten_sample` を利用可。

### 5-3. 基本情報

- 住所、電話、営業時間、定休日、アクセス、地図を整理して表示する。
- 不明情報は断定しない（「店舗へご確認ください」を使用）。
- 電話番号は `href="tel:..."` を必ず設定する。

### 5-4. CTA

- 問い合わせフォームより来店・予約・注文導線を優先する。
- モバイル下部に固定CTAを設置する。
- `href="#"` は使用しない。未確定時はページ内誘導にする。

### 5-5. 軽量化

- 画像最適化（適切サイズ、`loading="lazy"`、必要なら `srcset`）。
- 不要なアニメーションを避け、体感速度を優先する。

---

## 6. デザインレパートリー運用

毎回、以下4系統から最低1つを選び、可能なら別系統を掛け合わせる。

- 系統A: 職人・技術・信頼
- 系統B: 温かみ・親しみ・地域密着
- 系統C: 圧倒的差別化・モダン
- 系統D: 親切・わかりやすさ

直近案件と比較し、配色・レイアウト・装飾・見出しトーンの重複を避ける。

---

## 7. タイポグラフィ・改行ルール

- 対象は主に `h1`/`h2`/`h3` と短いキャッチコピー。
- 1文字だけの行落ちを避ける。
- 句読点・意味区切りで自然に改行する。
- 長文本文に過度な `span` 分割を適用しない。
- 見出しはやや控えめサイズを基本とし、`clamp()` 上限を抑える。
- 1320px超の大画面でも見出しが急拡大しないよう調整する。

---

## 8. ライティングルール

- プレースホルダー文言は禁止。
- 小田原商圏を意識した実用コピーを書く。
- 強み・体験価値・利用シーン・来店動機を具体的に書く。
- 数値や実績は裏取りできないものを捏造しない。
- 会社名略称は正式表記へ補正する。
  - `(株)` / `（株）` -> `株式会社`
  - `(有)` / `（有）` -> `有限会社`
  - `(合)` / `（合）` -> `合同会社`

---

## 9. Git・ブランチ運用

- 新規制作は必ず別ブランチで実施する。
- ブランチ名は `cursor/<企業名や業種>-landing-89fa` を使う。
- 完了後は commit / push / PR作成（または更新）を必須とする。
- HP制作PRは、特別依頼がない限り `index.html` のみ差分とする。

---

## 10. 検証チェックリスト

- `index.html` が存在する
- Tailwind CDNが含まれる
- 1ファイルHTML完結
- スマホ表示で崩れない
- 飲食店必須セクションが揃っている
- コンテンツ量が十分で薄くない
- 予約/来店導線が機能する
- Googleマップ埋め込みがある
- Instagram導線が機能する
- HTML構文破綻がない
- プレビューで表示確認済み

---

## 11. プレビュー運用

```bash
python3 -m http.server 8000 --bind 0.0.0.0
```

PHPが必要な場合のみ:

```bash
php -S 0.0.0.0:8000
```

Cloud環境では `localhost:8000` を直接開けない場合があるため、  
Ports / Preview / Open in Browser から 8000番ポートを開く。
